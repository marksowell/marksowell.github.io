# Security Policy

This repository powers `https://marksowell.com`.

For the machine-readable disclosure policy, see `https://marksowell.com/.well-known/security.txt`.

## Reporting a Vulnerability

If you believe you have found a security issue affecting this site or its supporting infrastructure, please report it privately by email at [mark@marksowell.com](mailto:mark@marksowell.com).

Please do not open a public GitHub issue for security reports.

## What to Include

- The affected page, URL, or component
- Clear reproduction steps
- Expected vs. actual behavior
- Potential impact
- Screenshots, logs, or proof-of-concept details if they help explain the issue

## Response Expectations

- Reports are typically acknowledged within 72 hours
- An initial triage update is typically provided within 7 days

## Scope

This policy covers the public website and repository-backed content served from `marksowell.com`.

Issues in third-party services, platforms, or products linked from this site should be reported to their respective owners.
